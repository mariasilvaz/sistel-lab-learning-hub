# Registro de cambios

Todas las integraciones relevantes a `main` se registran aquí: qué módulo se agregó o modificó, en qué fecha y quién lo desarrolló. Sigue el espíritu de [Keep a Changelog](https://keepachangelog.com/es-ES/1.1.0/): cada entrada es legible para alguien que no participó en el desarrollo.

El comité técnico actualiza este archivo como parte de la integración de cada *pull request* aprobado (ver `GOVERNANCE.md`, "Flujo de vida de un módulo").

## Formato de una entrada

```
## [Nombre del módulo o tema] - AAAA-MM-DD

### Agregado
- Qué se agregó (funciones, prácticas, documentación).

### Cambiado
- Qué se modificó en algo que ya existía.

### Corregido
- Qué error se corrigió.

**Desarrollado por:** nombre(s) del/los estudiante(s)
**Asesor:** nombre del docente asesor
**Pull request:** #NN
```

## Sin publicar

### Agregado
- Estructura base del repositorio: `README.md`, `CONTRIBUTING.md`, `GOVERNANCE.md`, `AGENTS.md`, `CODE_OF_CONDUCT.md`, `LICENSE` (MIT) y `LICENSE-DOCS.md` (CC BY 4.0).
- Estándares del proyecto en `docs/estandares/`: código, documentación y pedagógico.
- Plantillas de *Issues* y *pull request* en `.github/`.
- Archivo `CODEOWNERS` con el esquema inicial de revisión por módulo.

---

*Cuando se integre el primer módulo, agrega una nueva sección arriba de esta con el formato indicado.*

# Código de conducta

Este proyecto reúne a estudiantes y docentes de distintos niveles de experiencia, trabajando en equipos que pueden cambiar cada semestre. Este código de conducta busca que esa colaboración sea respetuosa y productiva para todas las personas involucradas.

## Se espera de todas las personas que participan

- Dar y recibir retroalimentación técnica de forma directa pero respetuosa. Señalar un error en el código o en una guía de práctica es parte normal del proceso de revisión; hacerlo de forma humillante o descalificante no lo es.
- Reconocer que quienes revisan un *pull request* (asesores, comité técnico) lo hacen como parte de su rol de acompañamiento, no como un obstáculo arbitrario.
- Ser paciente con quienes tienen menos experiencia en Git, GitHub o MATLAB — el proyecto asume que se aprende sobre la marcha.
- Dar crédito adecuado por el trabajo de otros (ver también `LICENSE-DOCS.md` sobre atribución).
- Usar los canales del proyecto (*Issues*, *pull requests*, revisiones) para discusiones sobre el trabajo; llevar los desacuerdos personales fuera de estos espacios.

## No se tolera

- Comentarios despectivos, sarcásticos o humillantes sobre el trabajo o la persona, en *Issues*, *pull requests*, o cualquier canal asociado al proyecto.
- Acoso, discriminación o trato irrespetuoso por motivos de género, origen, discapacidad, orientación, edad, nivel de experiencia o cualquier otra condición.
- Apropiarse del trabajo de otra persona sin atribución.
- Usar el acceso de escritura otorgado para un módulo con fines distintos a los previstos (ver `GOVERNANCE.md`).

## Alcance

Este código aplica a toda interacción dentro del repositorio (*Issues*, *pull requests*, revisiones de código, comentarios) y a las reuniones de seguimiento entre estudiantes y asesores relacionadas con el proyecto.

## Cómo reportar una situación

Si alguien no está siguiendo este código de conducta, repórtalo al coordinador académico del proyecto. Si la situación involucra directamente al coordinador académico o a un integrante del comité técnico, repórtalo al jefe del Departamento de Telecomunicaciones. Los reportes se manejan con la confidencialidad que la situación amerite.

## Consecuencias

El comité técnico y el coordinador académico pueden, según la gravedad de la situación:

- Solicitar una corrección del comportamiento.
- Retirar temporalmente el acceso de escritura al repositorio.
- Escalar el caso a las instancias disciplinarias correspondientes de la universidad, cuando el caso lo amerite.

## Reconocimiento

Este documento está inspirado en principios de códigos de conducta ampliamente usados en proyectos colaborativos de software (como el Contributor Covenant), adaptados al contexto académico específico de este proyecto.
